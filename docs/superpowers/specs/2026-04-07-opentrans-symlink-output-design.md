# OpenTrans 软链接与翻译输出目录设计

## 目标

修复项目目录为软链接时偶发无法打开的问题，并调整翻译行为与输出目录策略，使其更符合本地 macOS 使用场景。

## 需求范围

1. 统一处理项目根目录为软链接、目录内存在软链接文件/目录时的扫描与初始化行为。
2. 点击 Markdown 文件时只打开，不自动触发翻译；翻译必须由用户显式触发。
3. 默认翻译输出目录从“源项目同级目录”调整为“源项目内的下一级隐藏目录”。
4. 用户可以在设置中选择自定义翻译输出目录。

## 设计

### 项目目录解析

主进程继续保存两个路径：

- `selectedDir`：用户选择的原始路径，仅用于展示和回显。
- `srcDir`：真实目录路径，后续扫描、哈希、翻译任务都基于这个路径执行。

目录遍历继续跟随有效软链接，但通过真实路径集合避免循环递归。

### 翻译输出目录

新增输出目录解析规则：

- 默认模式：`<srcDir>/.opentrans/<basename(srcDir)>-translator`
- 自定义模式：`<customOutputDir>/<basename(srcDir)>-translator`

这样可以保证：

- 默认输出落在项目内部下一级目录。
- 多个项目共用自定义根目录时仍按项目名隔离。

### 交互行为

- 文件树点击：只打开原文与译文。
- 单文件翻译：通过文件悬浮按钮显式触发。
- 全量翻译：保留工具栏按钮，由用户主动触发。

## 风险控制

1. 输出目录改为项目内部后，扫描 Markdown 时必须跳过 `.opentrans`，避免把生成内容当成源文档重新纳入项目树。
2. 自定义输出目录为空时应回退到默认模式，避免写入非法路径。
3. 旧工作区迁移不做自动搬迁，只按新规则为新初始化路径建目录。
